# Recipe Management

## Overview

Users can create, organise, and consult their personal recipes within Famappy.

---

## Requirements

### RCP-01 — Create a recipe

**Description:** A user can create a new recipe by providing a name, a list of ingredients with quantities, and step-by-step instructions.

**Acceptance Criteria:**
- [ ] User can enter a recipe name
- [ ] User can add one or more ingredients, each with a name, quantity, and unit
- [ ] User can write preparation steps
- [ ] The recipe is saved and appears in the recipe list

**Status:** 📋 Defined

---

### RCP-02 — Edit a recipe

**Description:** A user can modify an existing recipe.

**Acceptance Criteria:**
- [ ] User can update the recipe name, ingredients, and steps
- [ ] Changes are persisted after saving

**Status:** 📋 Defined

---

### RCP-03 — Delete a recipe

**Description:** A user can permanently remove a recipe.

**Acceptance Criteria:**
- [ ] User is prompted to confirm before deletion
- [ ] Recipe no longer appears in the list after deletion

**Status:** 📋 Defined

---

### RCP-04 — Browse and search recipes

**Description:** A user can browse all their recipes and search by name.

**Acceptance Criteria:**
- [ ] All saved recipes are listed
- [ ] User can filter the list by typing a recipe name

**Status:** 📋 Defined
